# slint-poc（使い捨て検証用）

Tauri/WebView2 で再発する「縦サブモニター・混在DPIでの崩れ／オーバーレイ切替で main 消失」を、
**Rust ネイティブな Slint** なら素直に解決できるかを見極めるための最小 PoC。
engine/capture には依存せず、データはすべてモック。

## 実行

```powershell
# 初回はビルドに数分（Slint + femtovg を含む依存を取得・コンパイル）
cargo run --manifest-path slint-poc/Cargo.toml
```

2 つのウィンドウが出る:

- **DPS（枠なし・半透明・最前面）** … ヘッダ左の「⠿ drag」部分を掴んでドラッグ移動。背景は
  半透明（背後が透ける）。下部に winit が認識したモニタ一覧（座標・スケール）を表示。
  ヘッダの操作ボタン:
  - **Buffs: ON/OFF** … バフオーバーレイの表示/非表示（ネイティブ `hide()`/`show()`）。
  - **Lock: ON/OFF** … バフ窓を ON=クリックスルー（ゲーム操作優先・移動不可）／
    OFF=当たり判定あり（黄枠表示・**全面ドラッグで移動可**）に切替。
  - **Quit** … 終了。
- **Buffs（枠なし・透明・最前面）** … 円形＝バフ、棒＝デバフのタイマーが周期的に減る。
  既定はクリックスルー。移動したいときは DPS 窓の **Lock: OFF** にしてからドラッグ。

ウィンドウ位置・サイズは `%APPDATA%\bpsr-checker-slint-poc\window_layout.json` に自動保存され、
次回起動時に復元される（保存値が現在のモニタ範囲外なら既定位置へフォールバック）。

## 何を確認するか（採否判断のチェックリスト）

| # | 観点 | 期待 |
|---|---|---|
| 1 | 2 窓表示・透明・最前面 | 単一モニターで両窓が出て、Buffs が透けて最前面 |
| 2 | 複数モニタ配置 | Buffs が**セカンダリ**へ、DPS が**プライマリ**へ初期配置される |
| 3 | **縦型サブモニター** | DPS を縦サブへドラッグ→再起動→**同じ物理位置に復元**。崩れ/縮みなし |
| 4 | **混在DPI**（例: 100%+150%） | 文字サイズ・座標・透明合成が崩れない。起動ログのモニタ scale と実挙動が一致 |
| 5 | クリックスルー | Buffs の上をクリックしても背後（デスクトップ/ゲーム）に届く |
| 6 | 最前面 | ゲームを**ボーダーレス全画面**にしても両窓が上に出る（排他的フルスクリーンは方式不問で不可） |
| 7 | 画面外復帰 | サブモニターを外して再起動→ウィンドウが画面外に消えずプライマリへ戻る |
| 8 | **オーバーレイ切替で main が消えないか** | **Buffs: ON/OFF を連打**しても DPS 窓が白紙化しない（Tauri/WebView2 の現象が Slint で起きないかの核心検証） |

起動時に標準エラーへモニタ geometry を出力する（採否メモ用にコピーしておく）:

```
[poc] detected 2 monitor(s):
  - \\.\DISPLAY1   pos=(    0,    0) size= 2560x1440  scale=1.00 [primary]
  - \\.\DISPLAY2   pos=( 2560, -300) size= 1080x1920  scale=1.50
```

## 判断の指針

- 3〜5 が現行 Tauri 版より明確に安定するなら → 本実装（Slint 置換）を検討する価値あり。
- 差が出ない / 同様に崩れるなら → WebView2 起因ではない可能性が高く、Tauri 側の DPI・配置ロジック
  深掘りに戻す。

## 設計メモ（実装のキモ）

- 透明・skip-taskbar は `Backend::builder().with_window_attributes_hook(...)`（生成時に全窓共通で付与）。
  窓ごとの「実際に透けるか」は各 `.slint` の `background` で決める。
- 最前面・枠なしは `.slint` の `always-on-top` / `no-frame` プロパティ。
- クリックスルー・モニタ列挙は `WinitWindowAccessor::with_winit_window(...)` で winit へ降りて
  `set_cursor_hittest` / `available_monitors`。
- 全ウィンドウが**単一イベントループ**で回る（`slint::run_event_loop`）。WebView2 の別窓/別プロセスに
  起因する合成バグの温床を構造的に持たない。
- 円形ゲージの弧は Rust 側で SVG パスを算出（`mock::arc_commands`）し、`.slint` 内では数学関数を使わない。
